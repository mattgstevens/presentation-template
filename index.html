Online payments are largely centered around subscription model

This makes much sense for services, but what about access to content?

When in doubt of monetization, put up some advertising

  • Public awareness of information sharing via ad networks is rising
  • Browser add-ons for Chrome and Firefox to block third party scripts
  • Browsers that are purpose built to block trackers / be anonymous
  • Frustration of being interrupted to watch an ad

Momona

A scheme for payment to content creators.

  • Read online articles
  • Access to whitepapers and other scientific journals
  • Plug-in to CMS's

Not just for media companies

  • Pay per view for screen casts and tutorial
  • Once a threshold of total payments is reached content is unlocked
  • Donations and tips for a website

Individuals can be supported by their audience

Why Bitcoin?

Transaction fees are much smaller than credit cards

Transaction appears instantly on the blockchain

Immediate registration for creators: give a valid public address and you are done

Fast payments from viewer: receive a bitcoin URI that will open in a wallet app

Key points about Bitcoin

  • Bitcoin wallets contain keypairs, not coins
  • Public and private key pairs
  • Public key is the payable address
  • Private key is used to sign transactions that spend funds from the address
  • When sending a transaction, all value from the address must be sent (so called unspent transaction outputs or UXTO)
  • Coins are stored on the blockhain in the form of transaction-outputs

Web Client

  • Small footprint, is configured with a contentId
  • Stores the Invoice bitcoin address for future access
  • Returned content is rendered into page: string payload for content; with CORS permissions the web client can render video / images

Lessons Learned

  • Need secure transactions service: a protected server that has private keys to sign transactions
  • How to keep secrets safe: multi-signature which is an M of N scheme (aka Shamirs Shared Secrets) to sign transactions and grant access to sensitive commands
  • Hierarchical Deterministic Wallets: a seed is used to create first level of tree structure; parent keys can derive children, possible to derive public keys for receive only services
  • Additionally, HD wallets can represent an organizations departments or accounting category
